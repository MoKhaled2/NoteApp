# Project Structure & File Plan

This document maps out the current structure of the application and the purpose of each file.

## Core Layer (`lib/core`)
Shared utilities and configuration used across the app.

- **Router**:
    - [app_router.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/core/router/app_router.dart): Defines the GoRouter configuration and navigation paths.

- **Theme**:
    - [app_theme.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/core/theme/app_theme.dart): Defines the light and dark themes (colors, typography).
    - [theme_provider.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/core/theme/theme_provider.dart): Riverpod provider to manage and toggle theme state.

- **Utils**:
    - [delete_dialog.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/core/utils/delete_dialog.dart): Reusable dialog for confirming deletions.

## Feature: Auth (`lib/features/auth`)
Handles user authentication (Login/Signup).

- **Data**:
    - [auth_repository.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/auth/data/auth_repository.dart): Interface and implementation of Firebase Authentication.
    - [mock_auth_repository.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/auth/data/mock_auth_repository.dart): Mock implementation for testing/offline dev.

- **Domain**:
    - [app_user.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/auth/domain/app_user.dart): Data model representing a user.

- **Presentation**:
    - [login_screen.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/auth/presentation/login_screen.dart): UI for user login. **(Recently Revamped)**
    - [signup_screen.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/auth/presentation/signup_screen.dart): UI for user registration. **(Recently Revamped)**

## Feature: Notes (`lib/features/notes`)
The core functionality: creating, listing, and editing notes.

- **Data**:
    - [notes_repository.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/data/notes_repository.dart): Repository for Firestore interactions (CRUD + Sorting).
    - [mock_notes_repository.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/data/mock_notes_repository.dart): Mock repository for UI testing.

- **Domain**:
    - [note_model.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/domain/note_model.dart): Data model for a Note (fields: title, content, color, dates, etc.).

- **Presentation**:
    - [notes_list_screen.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/presentation/notes_list_screen.dart): Main dashboard showing the grid/list of notes.
    - [note_editor_screen.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/presentation/note_editor_screen.dart): Screen to create or edit a note.
    - [note_search_delegate.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/presentation/note_search_delegate.dart): Logic for the search bar.
    - **Widgets**:
        - [staggered_note_card.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/presentation/widgets/staggered_note_card.dart): Individual note card UI component.
        - [note_color_picker.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/features/notes/presentation/widgets/note_color_picker.dart): Widget for selecting note colors.

## Root (`lib/`)
- [main.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/main.dart): Entry point, sets up Providers and Routing.
- [firebase_options.dart](file:///c:/Users/MF/.gemini/antigravity/scratch/notes_app/lib/firebase_options.dart): Firebase configuration generated by CLI.
